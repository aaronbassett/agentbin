# agentbin

A file sharing service that lets AI agents publish rendered documents at public URLs.

## Architecture

Cargo workspace with three crates:
- `crates/core/` — Shared domain logic (`agentbin-core`)
- `crates/cli/` — CLI binary (`agentbin`)
- `crates/server/` — Web server binary (`agentbin-server`)

## Tech Stack

- **Language**: Rust (Edition 2021, latest stable MSRV)
- **Async runtime**: tokio
- **Web framework**: Axum (server)
- **CLI framework**: clap (derive API)
- **HTTP client**: reqwest (CLI)
- **Error handling**: thiserror (library), anyhow (binaries)
- **Serialization**: serde + serde_json
- **Auth**: ed25519-dalek (Ed25519 signing)
- **Rendering**: comrak (markdown), syntect (syntax highlighting)
- **Templates**: askama (compile-time HTML)
- **Logging**: tracing + tracing-subscriber
- **Linting**: clippy with pedantic, `#![deny(unsafe_code)]`
- **Formatting**: rustfmt (default config)
- **Storage**: Filesystem with JSON sidecars (no database)
- **Deployment**: fly.io (server)
- **Release automation**: release-plz (crates.io + tags) + cargo-dist (binaries + GitHub Releases + Homebrew)

## Common Commands

```sh
# Build
cargo build --workspace

# Test
cargo test --workspace

# Lint (must pass with zero warnings)
cargo clippy --workspace --all-targets -- -D warnings

# Format check
cargo fmt --all -- --check

# Build single crate
cargo build -p agentbin-core
cargo build -p agentbin
cargo build -p agentbin-server

# Preview release artifacts
dist plan
```

## Key Rules

- No `.unwrap()` or `.expect()` in production code — use proper error handling
- `#![deny(unsafe_code)]` in all crates
- Conventional commits required: `feat:`, `fix:`, `docs:`, `chore:`, etc.
- Each crate must compile and test independently
- Cross-crate deps flow inward (binaries → core, never reverse)
